## CyNER returns empty entities[] for ALL CVE descriptions

ran the fine-tuned `AI4Sec/cyner-xlm-roberta-large` checkpoint over all 323k
CVE descriptions via 50-shard SLURM array. every single record came back with
entities=[].

debugging log:
- shard 0 sample: "A vulnerability in the foo of bar allows X to do Y" -> []
- shard 23 sample: same pattern -> []
- spot-checked with HuggingFace inference widget on raw threat report sentence
  ("APT29 deployed Cobalt Strike via spearphishing") -> entities found OK

root cause: CyNER was trained on narrative threat report text. CVE descriptions
are terse, structured, and don't contain the surface forms (Threat-Actor,
Malware, Campaign) that the NER head learned to project.

next steps: try GLiNER (zero-shot). if same pattern -> pivot to structured
field extraction (CWE -> ATT&CK mapping table) instead of NER for CVEs.
NER stays useful for the threat report PDF chunks (narrative text).
